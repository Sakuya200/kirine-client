# Kirine Client

Kirine Client 是 Kirine（桐音）音频工作台的桌面客户端，面向本地语音生成、声音克隆和模型训练场景，提供统一的图形界面来管理模型、说话人和历史任务。

## 重要提示

当前项目尚未进入正式版本，功能仍在持续调整中，整体稳定性和兼容性都还没有完全固化。

- 各版本之间不一定兼容。
- 更新后可能导致旧数据、旧配置或旧运行环境无法继续使用，严重时可能直接导致程序不可用。
- 如果遇到升级后无法启动、任务异常、模型状态错乱、历史记录异常等问题，请及时反馈，我们会第一时间处理。

## 这是什么

Kirine Client 目前以本地模式运行，主要提供以下能力：

- 模型管理：查看支持的基础模型，执行安装和卸载。
- 文本转语音：输入文本并选择说话人生成音频。
- 声音克隆：上传参考音频和参考文本，生成目标文本语音。
- 模型训练：导入样本或数据集，训练新的说话人模型。
- 说话人管理：查看、筛选、编辑和删除说话人。
- 历史任务：统一查看训练、TTS、声音克隆任务及结果。
- 设置：管理目录、硬件类型和注意力实现等运行参数。

## 当前支持的基础模型

支持模型与预置说话人由根目录 [supported_models.json](supported_models.json) 定义，应用启动后会自动同步到本地数据库。

| 基础模型 | 规格 | 当前能力 |
| --- | --- | --- |
| `qwen3_tts` | `1.7B`、`0.6B` | 文本转语音、声音克隆、模型训练 |
| `vox_cpm2` | `2B` | 文本转语音、声音克隆、模型训练、LoRA 参数链路 |
| `moss_tts_local` | `1.7B` | 文本转语音、声音克隆、模型训练 |

## 首次使用前

1. 当前主要面向 Windows 10/11。
2. 首次运行或首次使用某个模型时，准备时间可能较长，因为会涉及依赖初始化、模型检查和运行时准备。
3. 训练任务更适合在有 GPU 的环境下使用；CPU 也可以运行，但速度会明显慢很多。
4. 如果只是修改目录、硬件类型或注意力实现，优先在设置页中修改，不建议频繁手工编辑配置文件。

## 快速上手

建议按下面的顺序使用：

1. 打开设置页，确认数据目录、日志目录、模型目录是否正确。
2. 打开模型管理页，先安装需要使用的基础模型。
3. 如果要做文本转语音，先确认已有可用说话人。
4. 如果要训练自己的说话人，先准备音频样本和标注文件，再到模型训练页创建任务。
5. 所有任务的结果和状态都可以在历史任务页统一查看。

## 页面说明

### 模型管理

- 查看当前支持的模型与规格。
- 安装或卸载模型。
- 模型安装完成后，相关能力会自动出现在对应任务页面。

### 文本转语音

适合已经有可用说话人的场景。

基本步骤：

1. 选择基础模型、模型规格和说话人。
2. 选择语言和输出格式。
3. 输入文本内容。
4. 提交任务并等待生成完成。

补充说明：

- 只有状态为 `ready` 且基础模型匹配的说话人会出现在候选列表。
- 结果会进入历史任务，可试听、导出和再次执行。

### 声音克隆

适合根据一段参考音频生成目标文本语音。

基本步骤：

1. 选择基础模型、规格、语言和输出格式。
2. 上传参考音频。
3. 填写参考文本。
4. 填写目标文本。
5. 提交任务并等待完成。

当前支持的参考音频格式：

- `wav`
- `mp3`
- `flac`
- `ogg`

### 模型训练

适合训练自己的说话人模型。

基本步骤：

1. 选择语言、基础模型和模型规格。
2. 填写模型名称和描述。
3. 设置训练参数。
4. 导入单样本或带标注的数据集。
5. 启动训练并在历史任务页跟踪状态。

当前支持的导入格式：

- 音频：`wav`、`mp3`、`flac`、`ogg`
- 标注：`jsonl`、`xlsx`、`xls`

### 说话人管理

- 查看说话人总数、可用数量、训练中数量和样本总量。
- 按关键词、语言和状态筛选。
- 查看详情，编辑名称和描述。
- 删除不再需要的用户说话人。

### 历史任务

- 统一查看训练、文本转语音、声音克隆任务。
- 查看任务详情和结果。
- 导出输出文件。
- 将历史参数回填到页面中再次执行。

### 设置

- 配置 `data_dir`、`log_dir`、`model_dir`。
- 配置 `hardware_type` 和 `attn_implementation`。
- 当前产品形态仍以本地模式为主。

## 配置文件

根目录 [config.toml](config.toml) 是当前应用主配置文件，常见字段如下：

```toml
[basic]
mode = "local"
data_dir = 'D:\Project\temp\kirine-client\data'
log_dir = 'D:\Project\temp\kirine-client\logs'
model_dir = 'D:\Project\temp\kirine-client\models'

[training]
hardware_type = "cpu"
attn_implementation = "sdpa"
```

- `data_dir`：任务数据、样本和数据库默认存放目录。
- `log_dir`：应用日志和任务日志目录。
- `model_dir`：本地模型目录。
- `hardware_type`：运行硬件类型，例如 `cpu` 或 `gpu`。
- `attn_implementation`：注意力实现方式，默认使用 `sdpa`。

## 常见问题

### 第一次运行为什么很慢

首次运行或首次调用某个模型时，可能会进行依赖初始化、模型检查、运行时准备或资源复制，所以等待时间通常会明显更长。

### 任务长时间没有完成怎么办

建议按下面顺序排查：

1. 先看页面顶部通知和任务状态是否仍在刷新。
2. 确认当前是否处于首次安装、首次推理或首次训练阶段。
3. 如果刚修改过目录设置，先按提示重启应用。
4. 查看 `log_dir` 下的任务日志，常见路径形如 `task\{任务类型}_{任务id}.log`。
5. 如果模型安装状态异常，先到模型管理页检查状态是否一致。

### flash-attn 能直接用吗

当前 Windows 环境下通常缺少稳定可用的官方支持，默认仍建议使用 `sdpa`。如果你确实需要 `flash-attn`，一般需要自行准备兼容环境。
