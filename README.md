# Kirine Client

Kirine Client 是 kirine（桐音）音频工作台的桌面客户端，本质上是音声模型（目前仅支持Qwen3-TTS）的客户端UI层实现，具备文本转语音，声音克隆，说话人模型训练，说话人管理等功能，通过易用的UI简化模型操作。项目采用 Tauri 2 作为桌面容器，前端使用 Vue 3 + TypeScript，后端使用 Rust，并集成一套本地 Python 模型运行时。

这个项目的目标：
- 更友好的操作UI
- 更简单的开箱即用
- 更多样化的音频生成功能
- 更强大的多模型兼容能力

## 功能概览

### 1. 文本转语音

- 输入文本并选择说话人，创建文本转语音任务。
- 支持任务结果读取、音频回放与导出。
- 本地推理链路由 Rust 调用 Python 脚本执行。

### 2. 声音克隆

- 上传参考音频与参考文本，生成目标语音。
- 任务结果会进入历史记录，便于重复试听和导出。

### 3. 模型训练

- 支持单样本或数据集方式提交训练任务。
- 提供训练标注模板导出能力，方便准备 JSONL 或 Excel 数据。
- 本地训练流程包含运行时准备、基础模型下载、音频编码预处理与训练执行。

### 4. 说话人管理

- 提供说话人信息的新增、查询、编辑、删除能力。
- 为文本转语音和声音克隆提供统一的角色数据来源。

### 5. 历史任务管理

- 统一查看模型训练、文本转语音、声音克隆任务。
- 支持查看任务详情、读取输出文件、保存结果到本地、删除历史记录。

### 6. 设置与运行环境管理

- 配置本地数据目录、日志目录、模型目录。
- 配置远端服务地址和令牌。
- 配置模型运行相关参数，例如 attention implementation。

## 技术栈

- 桌面框架：Tauri 2
- 前端：Vue 3、TypeScript、Vue Router、Pinia
- UI：Tailwind CSS、Headless UI、Heroicons
- 后端：Rust
- 模型运行时：Python

## 使用教程

### 使用前提示

1. 程序安装包本身前往release页面即可下载，安装时可以选择安装路径，安装过程到最后时留意系统安全提醒，如果有提醒的话通过即可，是安装程序在执行资源解压的操作
2. **由于目前项目暂未进入正式版本，部分功能实现尚未稳定，更新后可能会导致严重的 BUG 甚至无法使用，如遇到问题请及时反馈，我们会尽快处理**
3. 在第一次使用功能时需要等待较长的时间，因为涉及到模型相关依赖的下载。当前 Qwen3-TTS 本地运行默认会准备 `Qwen3-TTS-12Hz-1.7B-Base`、`Qwen3-TTS-Tokenizer-12Hz` 和 `Qwen3-TTS-12Hz-1.7B-CustomVoice` 三个模型，后续的任务会快很多。如果想查看执行情况可以前往查看对应任务的日志（logs\task\{任务类型}_{任务id}.log）
4. 非必要不要随意修改配置文件内容，如需修改数据，模型等目录的路径，注意力机制算法请前往配置页面修改
5. 目前程序只支持Win10/11系统，Mac版本请等待后续兼容，并且由于目前程序只支持本地运行，服务端版本正在筹备中，如果需要使用模型训练这种大规模计算场景，对PC端的显卡要求会较高，请确保显存尽量在8G以上再使用，其他的场景（比如声音克隆）对显卡的要求不高，甚至也可以直接使用CPU运行


### 文本转语音

文本转语音页面适合在已有可用说话人的前提下快速生成音频。

#### 文本转语音操作步骤

1. 在说话人下拉框中选择一个状态为“可用”的说话人。
2. 选择硬件类型、语言和输出格式。
3. 在文本框中输入要合成的文本内容。
4. 如有需要，在声音 Prompt 中补充风格描述；不填则使用默认风格。
5. 点击生成，等待任务完成。

#### 结果会保存到哪里

- 结果会写入本地数据库。
- 结果文件会写入本地输出目录。
- 生成完成后可以直接播放、下载，也可以在历史任务里再次查看。

#### 文本转语音关注点

- 当前页面只会显示状态为 `ready` 且与当前基础模型一致的说话人；如果下拉框为空，先检查说话人数据和基础模型选择是否正确。
- 文本越长、段落越多，生成耗时通常越高。
- 输出格式可选 `WAV`、`MP3`、`FLAC`，其中无损格式体积更大。
- 历史任务支持“再次执行”，可以把旧任务参数重新带回当前页面，减少重复填写。

### 声音克隆怎么使用

声音克隆用于根据参考音频和参考文本生成新的目标语音。

#### 声音克隆操作步骤

1. 选择基础模型和硬件类型。
2. 选择语言与输出格式。
3. 上传参考音频文件。
4. 填写参考文本，也就是参考音频对应的真实台词。
5. 填写目标文本，也就是你希望克隆后读出的内容。
6. 点击生成并等待完成。

#### 当前支持的参考音频格式

- `wav`
- `mp3`
- `flac`
- `ogg`

#### 声音克隆关注点

- 参考音频和参考文本尽量保持严格一致，否则克隆结果可能出现发音不稳或韵律异常。
- 首次在某个硬件类型下执行声音克隆时，通常会涉及运行时准备和基础模型下载。当前会准备 `Base`、`Tokenizer` 与 `CustomVoice` 三个目录，等待时间可能较长。
- 任务执行过程中，页面会自动刷新状态；生成完成后可以试听和保存结果。
- 历史任务中的声音克隆记录支持回填配置，便于重新生成类似结果。

### 模型训练

模型训练页面支持单样本导入和数据集导入混合方式。

#### 训练前建议

- 尽量准备语言一致、噪音较低的语音样本。
- 建议先裁掉明显静音段。
- 单条样本不宜过长，页面当前建议尽量控制在 15 秒以内。

#### 方式一：单样本导入

1. 选择训练语言、基础模型和硬件类型。
2. 填写模型名称、训练轮数、批大小。
3. 选择一个音频文件。
4. 填写这条音频对应的文本稿。
5. 点击“加入导入列表”。
6. 导入列表中至少有一条样本后，点击开始训练。

#### 方式二：数据集导入

1. 准备一个 ZIP 压缩包，里面放音频样本。
2. 准备一个标注文件。
3. 在页面上分别选择 ZIP 压缩包和标注文件，标注文件格式参考模板。
4. 点击“加入导入列表”。
5. 确认导入列表无误后开始训练。

#### 当前支持的输入格式

- 音频：`wav`、`mp3`、`flac`、`ogg`
- 标注：`jsonl`、`xlsx`、`xls`

#### 关于训练模板

- 页面支持下载训练标注模板。
- 模板可用于快速整理 JSONL 或 Excel 标注内容，适合首次准备数据时参考格式。

#### 训练关注点

- 首次训练通常是整个应用里等待时间最长的操作之一，因为它不仅要准备依赖和模型，还要做音频编码预处理与训练本身。
- 如果选择 CPU，训练速度会明显慢于 CUDA，首次准备也会更久。
- 导入列表只是当前任务的待提交内容，确认无误后再点击开始训练。
- 训练任务创建后会进入历史记录，后续可以从历史详情中再次带回配置。

### 说话人管理

说话人管理页主要用于维护现有说话人数据。

#### 说话人管理可执行操作

- 查看说话人总数、可用数量、训练中数量、样本总量。
- 通过关键词、语言、状态筛选说话人。
- 查看详情。
- 编辑名称和备注。
- 删除不再需要的说话人。

#### 管理关注点

- 文本转语音页面只会使用状态为“可用”且基础模型匹配当前选择的说话人。
- 删除说话人前，建议先确认它是否仍被历史任务或当前工作流依赖。
- 如果你刚完成训练但列表未更新，可以手动点击刷新列表。

### 历史任务

历史任务页用于统一查看模型训练、文本转语音和声音克隆记录。

#### 历史任务可执行操作

- 按任务类型、状态筛选。
- 按任务 ID、标题、说话人搜索。
- 打开详情查看完整参数和执行结果。
- 对已有任务执行“再次执行”，把历史参数带回对应页面。
- 删除不再需要的历史记录。

#### 历史记录关注点

- “再次执行”并不是直接后台重跑，而是先把参数带回目标页面，方便你确认后重新提交。
- 如果任务失败，可以先在详情里查看失败原因，再决定是否重新执行。
- 删除历史任务会同步逻辑删除关联详情记录，建议确认无误后再删除。

### flash-attn 相关
flash-attn 是一种具有I/O感知能力的快速且内存高效的精确注意力机制算法实现，在使用模型时使用这个算法可以有效降低显存占用以及计算效率
由于目前官方暂时没有支持windows实现，要想使用需要去社区寻找符合自己主机环境的对应编译版本，或者自行编译（非常耗时，一般需要好几小时起步），由于并不是所有用户的主机环境都有符合自己的编译版本，故本程序暂不支持该算法（即便可以选择），如有需要请自行在src-model下完成编译版本的安装

### 遇到长时间等待或者任务显示失败时怎么判断异常
通常可以按下面的顺序判断：

1. 看页面顶部是否仍显示“正在创建任务”“任务执行中”“正在保存配置与迁移目录”等提示。
2. 首次训练或声音克隆时，优先判断是否正处于依赖安装、模型下载或模型加载阶段。
3. 如果刚改过数据目录、日志目录、模型目录，先按提示重启应用，再继续操作。
4. 如果长时间无响应且没有任何状态更新，再去查看日志目录中的日志文件，或在项目群反馈问题。

## 开发者须知

### 环境要求

建议先准备以下环境：

- Node.js 与 npm，用于启动前端与 Tauri 开发流程。
- Rust 工具链，用于构建 Tauri 桌面端。
- Python 运行环境，用于本地模型推理、声音克隆和训练。
- Windows 环境下的 WebView2 运行时。

### 安装依赖

```bash
npm install
```

### 启动前端开发模式

适合只调试 Vue 界面：

```bash
npm run dev
```

启动后可在本地开发地址访问前端页面。

### 启动桌面客户端

适合联调 Tauri、Rust 与前端：

```bash
npm run tauri dev
```

这个命令会先启动前端开发服务器，再拉起桌面应用。

### 构建前端产物

```bash
npm run build
```

### 首次使用建议

1. 启动应用后，先进入设置页确认数据目录、日志目录、模型目录是否正确。
2. 如果使用本地训练或推理，确认 Python 运行环境和模型资源可用。
3. 先创建说话人资料，再进行文本转语音或声音克隆任务。
4. 训练任务完成后，可在历史页查看记录与导出相关结果。
5. 注意第一次启动前需要准备一个resources目录放到src-tauri下，作为打包时携带内容，包括ffmpeg-8.0.1-essentials_build.zip，sox-14.4.2-win32.exe，以及src-model目录的压缩包文件（可以通过使用src-model中自带的package_src_model.ps1脚本打包）

## 主要页面说明

- 模型训练：提交训练数据、设置训练参数、发起训练任务。
- 文本转语音：输入文本、选择说话人、生成并试听语音。
- 声音克隆：提供参考音频与目标文本，生成克隆语音。
- 说话人管理：维护说话人资料，供任务页面复用。
- 历史任务：查看任务记录、读取输出音频、导出结果。
- 设置：维护本地目录、远端接口和模型运行参数。

## 配置说明

项目根目录下的 config.toml 用于维护应用运行配置，当前主要包括：

- basic：运行模式、本地数据目录、日志目录、模型目录。
- remote：远端服务地址与访问令牌。
- training：基础模型准备状态与注意力实现配置。

默认配置偏向本地模式，适合在单机环境下调试和运行。

## 开发说明

- 前端路由已覆盖模型训练、文本转语音、声音克隆、说话人管理、历史任务、设置等页面。
- Tauri hooks 已提供设置、说话人、历史任务与任务创建相关命令入口。
- 本地模式下，Rust 服务层会负责配置加载、日志初始化、服务初始化和退出时资源关闭。
